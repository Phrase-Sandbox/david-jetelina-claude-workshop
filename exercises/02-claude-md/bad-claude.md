# Project Guidelines

* Please follow Terraform best practices when writing code
* Try to use variables instead of hardcoding values
* It would be great if all resources had proper tags
* We prefer using modules where possible
* Please don't forget to add outputs
* Try to keep the code DRY
* It's nice to have descriptions on variables
* Maybe add some validation blocks if you can
* We usually put providers in a separate file
* Please try to use consistent naming conventions
* Don't do anything dangerous
* Use comments to explain complex logic
* We like to use data sources instead of hardcoding ARNs
* Try to organize resources logically
* Please remember to add lifecycle blocks where appropriate
